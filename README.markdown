# 一头扎在书本里，狂看书
1. 建立知识的框架，在遇到问题时逐步完善局部的知识点，不要一头扎到书本里，书本只是工具，工具而已。避免啃书。要思考和阐述，技术是练出来的。练习是解决问题。学习技术看书要迅速，搞清楚概念，然后实践。专注于当下的技术工具，在遇到其他的技术技术性工具时，将其添加到下一个单元里去。如果上一个技术工具影响到了当前的技术的深入学习，那么就将当前的技术放下，初步掌握了影响这个技术工具的下一个技术工具，再回过头来继续这个技术工具。
